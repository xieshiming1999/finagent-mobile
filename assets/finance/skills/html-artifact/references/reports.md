# Report Patterns

## When to use

Recurring documents — status updates, post-mortems — benefit most from a bit
of structure and color. A small chart and a colored timeline turn something
people skim into something they actually read.

## Templates

### 11 — Weekly Status Report

**Shape:** Single-page dashboard with sections.

**Key elements:**
- Summary stats at top: shipped / in progress / blocked / total
- "Highlights" list with green checkmarks
- "Slipped" list with yellow warnings
- Small bar or sparkline chart showing velocity trend
- Footer: next week's focus, risks

**Adapt for:** sprint reports, project updates, team standups, investor
updates, personal weekly reviews.

### 12 — Incident Timeline

**Shape:** Vertical timeline with expandable entries.

**Key elements:**
- Minute-by-minute timeline (vertical line with circles)
- Each event: timestamp, description, severity badge
- Expandable log excerpts (code blocks)
- Impact summary box
- Follow-up checklist with checkboxes
- "Contributing factors" section

**Adapt for:** post-mortems, runbook execution logs, debug narratives,
security incident reports.
